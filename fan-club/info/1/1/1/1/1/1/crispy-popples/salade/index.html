Architecture Recovery Plan

Three steps to restore structural health and ensure scalability.

01. Flatten

Eliminate Nesting

Move crispy-popples up to the /fan-club/ directory. Delete the recursive 1/1/1 folders.

mv .../crispy-popples/* ./fan-club/
rm -rf ./info/1
02. Centralize

Convert to Markdown

Leverage Jekyll (native to GitHub Pages). Store content in _posts or docs folder as Markdown.

---
layout: post
title: Crispy Popples
---
03. Index

Generate Sitemap

Create a root index.html that links to all child pages, ensuring no content is orphaned.

<a href="/fan-club/popples">Go</a>
