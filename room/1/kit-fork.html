Méthode 1 GitHub Pages (recommandé)
Gratuit, permanent, accessible à tous. Votre arsenal en ligne en 5 minutes.
1
Créez un compte GitHub — github.com (gratuit). Si vous en avez déjà un, passez à l'étape 2.
2
Créez un nouveau repository — cliquez "New repository". Nom : demogarchie (ou le nom de votre choix). Public. Cochez "Add a README".
3
Uploadez tous les fichiers HTML — glissez-déposez les 17 fichiers dans le repository. Commit.
4
Activez GitHub Pages — Settings → Pages → Source : "Deploy from a branch" → Branch : "main" → Save.
5
Votre site est en ligne à l'adresse https://[votre-user].github.io/demogarchie/. Partagez le lien.
# Alternative : en ligne de commande
git clone https://github.com/[votre-user]/demogarchie.git
cp *.html demogarchie/
cd demogarchie
git add . && git commit -m "Deploy DÉMOGARCHIE" && git push
Méthode 2 Clé USB (offline / résilient)
Aucune connexion internet nécessaire. Fonctionne sur n'importe quel ordinateur avec un navigateur.
1
Téléchargez tous les fichiers HTML sur votre ordinateur.
2
Copiez-les sur une clé USB dans un dossier "DEMOGARCHIE".
3
Ouvrez index-demogarchie.html depuis la clé USB. Tout fonctionne localement.
Note : les polices Google Fonts nécessitent internet. Hors ligne, le navigateur utilisera des polices système de remplacement. Les outils fonctionnent à 100%.
Méthode 3 Diffusion par email
Chaque fichier HTML est autonome. Envoyez un outil spécifique à quelqu'un qui en a besoin.
1
Choisissez le module pertinent (ex: generateur-recours.html pour quelqu'un qui a un problème administratif).
2
Joignez le fichier HTML à un email. Le destinataire l'ouvre dans son navigateur.
3
Ajoutez index-demogarchie.html pour qu'il découvre les autres outils.
Hack Personnalisation
Chaque fichier HTML contient ses données directement dans le code JavaScript. Voici comment les modifier.
→
Données : ouvrez le fichier dans un éditeur de texte (VS Code, Notepad++, même le Bloc-notes). Les données sont dans des objets JavaScript nommés ROUTES, NARRATIVES, SOURCES, etc. Modifiez les valeurs.
→
Style : chaque fichier a ses variables CSS en haut (:root). Changez les couleurs, les polices, les tailles.
→
Contenu : le texte est en français belge. Traduisez en néerlandais, en allemand, ou adaptez pour un autre pays (les principes sont universels).
→
Localisation : ajoutez les données de votre commune, votre région. Les structures de données sont pensées pour être extensibles.
